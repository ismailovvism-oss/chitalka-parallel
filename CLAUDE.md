# CLAUDE.md — Параллельная читалка

Управляющий файл проекта для Claude Code. Читается автоматически. Здесь — рабочий
контекст, который иначе пришлось бы объяснять каждой новой сессии. Полная
спецификация формата — в `SPEC.md`; список идей — в `BACKLOG.md`.

> **Большая цель.** Эта читалка — слой просмотра будущей **библиотеки исламских знаний**
> (каталог, классификация, теги, полнотекстовый поиск, авторы). Целевая архитектура и
> решения (static-first с швами под бэкенд; контент отделён от приложения; сканы/PDF — в
> объектное хранилище) — в `ARCHITECTURE.md`. Развивая читалку, держи её совместимой с
> этим: стабильные id, состояние за абстракцией, формат Контракта как общий.

## Что это

Статическое веб-приложение (vanilla JS, без сборки и фреймворков) для
**параллельного чтения** книги на двух языках (оригинал + перевод), выровненных по
смысловым секторам. PWA (офлайн через `sw.js`). Запуск — любой статический сервер
(`python3 -m http.server 8000`), `file://` не годится из-за `fetch`.

Ключевые файлы:
- `index.html` / `style.css` — оболочка и стили.
- `app.js` — всё приложение (рендер, навигация, поиск, закладки, выделения, статистика, настройки).
- `parser.js` — **чистые функции Контракта без DOM**. Используется и в браузере, и в
  Node (`tools/validate.js`). НЕ добавлять сюда обращения к DOM — сломается валидатор.
- `books/` — контент. `books/index.json` — реестр; каждая книга в `books/<id>/`
  (`book.json` + папки языков). `books/_sample/` — эталонная мини-книга и тест формата.
- `tools/validate.js` — валидатор Контракта: `node tools/validate.js books/<id>`.
- `tools/import-tam.js` — импорт книги «Тауфик аль-Маннан» (см. ниже).

## Контракт (кратко; детали в SPEC.md)

- Сектор: `<!-- sNNN -->` перед абзацем. Текст-сектора **выровнены по id** между
  языками (`s012` в `ar` ↔ `s012` в `ru`). Сплит длинного: `s050a`, `s050b` склеиваются
  в базовый `s050`. Пустая строка завершает сектор.
- Страницы: `<!-- pN -->` **только в `ar/`**, протягиваются по id в `ru`.
- Сноски: `<!-- fnN -->` блоком в конце файла; ссылка в тексте — `[^N]` (только числа).
- Манифест `book.json`: `languages` (первый — оригинал/источник страниц), `rtl`,
  `chapters[{file,title}]`. Книга может быть **одноязычной** (`languages:["ru"]`).

### ⚠️ Сноски — ПЕР-ЯЗЫЧНЫЕ (не откатывать!)

Сноски **не спариваются** между языками. У каждого языка свой список и своя нумерация;
ссылка `[^N]` ведёт к сноске `fnN` **того же языка** (в DOM id `fnN@lang`). Причина:
авторские сноски-цитаты (в `ar`) и пояснения переводчика (только в `ru`) — разной
природы и числа, спаривать их по номеру неверно. В `parser.js` сноски складываются как
одноязычные пары с тегом `lang`; в `app.js` клик/инлайн/скачок/поиск и видимость
учитывают язык. Если будущая задача — «почему сноски не парные», это **сделано
намеренно**, см. коммит «Сноски — пер-язычные».

### Целиком односторонняя глава

Если в главе у одного языка 0 текст-секторов (второй язык ещё не подключён) — это не
ошибка: валидатор даёт одну строку «глава целиком только в <lang>», а не шум по каждому
сектору. Полезно для недопереведённых книг.

## Книга «Тауфик аль-Маннан» (`books/tawfiq/`)

Апологетический труд по акыде, арабский + русский. Контент **сгенерирован** из
отдельного репозитория и завендорен сюда (закоммичен). Источник:

- Репозиторий: `https://github.com/ismailovvism-oss/tawfiq-al-mannan` (отдельный, может
  быть вне MCP-скоупа — работать через `git clone` по https).
- Формат там — «Вычитка»: `source/<метка>.md` (арабский) + `translation/<метка>.md`
  (русский). **Секторы = абзацы по одной пустой строке, спарены по порядку**; аят
  (`۞арабский` + русский вплотную) = один сектор. Сноски — внизу файла.
- **Паритет** (равное число абзацев ar/ru) автор сводит в ветке **`split-paragraphs`**
  репозитория перевода. Импортировать надо именно с неё (в `main` правок может не быть).

### Как (пере)импортировать

```bash
git clone https://github.com/ismailovvism-oss/tawfiq-al-mannan.git /tmp/tam-book
cd /tmp/tam-book && git checkout split-paragraphs && git pull
cd <корень читалки>
node tools/import-tam.js /tmp/tam-book      # → books/tawfiq/{ar,ru}/ + book.json + index.json
node tools/validate.js books/tawfiq
```

Конвертер: именованные сноски (`[^ghuraba]`, `[^и1]`) → числовые `[^N]` + `<!-- fnN -->`;
вики-ссылки развёрнуты; нумерация сносок своя на каждый язык. **Заголовок не становится
отдельным сектором** — прицепляется к следующему текст-сектору как ведущая часть того же
`sNNN` (части `a`=заголовок, `b`=текст), иначе лишний блок сдвинул бы парность с арабским.
**Арабские строки двуязычных секторов** (аят `۞…`, арабский хадис) выкидываются из `ru/`
(остаются в `ar/`), иначе арабский виден дважды; сектор сохраняется. Маркеры страниц
мастера `<!-- ص: N -->` → `<!-- pN -->`; сканы страниц — в `books/tawfiq/img/` (`hasImages`).
**Арабский выдаётся только при точном посекторном паритете**; где ar≠ru — глава ru-only.

**Не редактировать руками** `books/tawfiq/**` — только регенерировать импортером. В
`source/` репозитория перевода НЕ писать (правило того проекта; там только автор/Вычитка).

### Текущий статус паритета

импортёр доработан). При переимпорте после новых правок — снова прогнать валидатор.

## Разработка и деплой

- Разработка — в ветке сессии (`claude/...`); по готовности **fast-forward в `master`**
  (это «опубликованная» ветка). НЕ пушить в чужие ветки без явного разрешения.
- **Бампать `VERSION` в `sw.js`** при изменении файлов оболочки (`index.html`, `app.js`,
  `parser.js`, `style.css`) — иначе PWA-клиенты не обновятся. Сейчас `chitalka-v14`.
- ⚠️ **Репозиторий разрабатывается с двух устройств параллельно.** ВСЕГДА `git fetch` +
  работа поверх свежего `origin/master` перед пушем; никогда не force-push. `master`
  «опубликованный». Бренд — **«Библиотека Алькасави»**.
- `parser.js` держать Node-совместимым (без DOM) — на нём валидатор.
- Проверять синтаксис: `node --check app.js && node --check parser.js`.

## Состояние фич (что уже есть)

v1 (базовое чтение/навигация/сноски/поиск/закладки), v2 (свайп глав, подсветка из
поиска, тема «как в системе», бэкап настроек, выравнивание текста), v3 (текстовые
выделения, статистика чтения, диапазоны страниц в оглавлении). Поддержаны одноязычные
книги, две колонки, RTL, гарнитуры/размеры на язык. **Отложено** (в BACKLOG): словарь по
тапу (нет источника), мультиперевод 3+ языков (ядро готово, нет контента).
